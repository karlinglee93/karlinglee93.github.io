IP 地址和端口号

两台电脑进行通信的时候，至少需要两个信息：IP 地址和端口号。
格式：IP 地址:端口号

IP 地址 - 计算机的标识

  1. 什么是IP 地址
    IP：Internet Protocol，互联网协议。
    由4个无符号的字节组成的一串数字。
    最小的IP：0.0.0.0；最大的IP：255.255.255.255

  2. 自己的IP 地址
    127.0.0.1

端口号 - 通信的窗口

  1. 端口的概念
    端口号：Port Number，可以让一台电脑同时传输多路数据。
    每台电脑都有65536个端口，范围是0 ～ 65535

  2. 端口的作用
    端口是数据传输的窗口。使用不同的端口传递数据的时候，彼此之间互不影响。

  3. 网络，IP 地址和端口号

Java TCP 编程

Java Socket 编程分为TCP 和UDP，都在java.net 这个包中

  1. TCP(Transmission Control Protocol/Internet Protocol)：传输控制协议/网间协议
  2. UDP(User Data Protocol)：用户数据报协议

都是数据传输协议，也是数据传输的方式。

TCP 的数据传输模式

在TCP 协议中，有服务端（Server 端）和客户端（Client 端）。
TCP 传输数据的过程：服务器端程序在本机的某个端口上监听，等待客户端连接到此端口，一旦客户端连接到了此端口，服务器端和客户端就可以进行数据传输了，数据传输结束后，关闭连接。

第一个TCP 小程序

Socket相关类，都在java.net 包中
InetAddress 代表IP 地址的类
ServerSocket 代表服务器端的类
Socket 代表客户端/连接的类

InetAddress 类

其构造方法是私有的，故我们不能通过new 关键字来创建一个InetAddress 类的实例。
但InetAddress 类有几个静态方法，可以用于生成InetAddress 的实例。

比如getByAddress(byte[]) 这个静态方法，其参数是一个byte 数组。
一个IP 地址是由4 个字节组成的，只要将这4 个字节顺序放在一个byte 数组中，然后将它作为参数传递给这个方法，就可以得到一个IP 地址。

ServerSocket 类

代表服务端。它有以下几个方法：

1. 构造方法ServerSocket(int)：参数是一个端口号，ServerSocket 的实例创建出来后，会在指定的端口监听。
2. accept()：在指定的端口等待客户端连接过来。在客户端连接过来之前，当前线程是处于挂起状态的，即如果没有客户端连接过来，这个方法不会执行结束。返回一个Socket 类型的实例。
3. close()：结束监听，释放资源。

Socket 类

可以看作是两个端口之间的连接。两台电脑之间的通信也使用这个类。

创建Socket 类实例的两种方式：

1. ServerSocket 类的accept() 方法会返回一个Socket 实例。
2. 通过Socket 类的构造方法创建。

主要的方法：

1. 构造方法Socket(InetAddress, int)：指定Socket 需要连接的远程IP 地址和端口。
2. getInputStream()：返回值是一个InputStream 的实例，通过输入流可以读取信息。
3. getOutputStream()：返回值是一个OutputStream 的实例，通过输出流可以向远程计算机输入信息。

UDP 是与TCP 并列的一种数据传输协议

UDP 中没有服务器端和客户端，有数据的发送端和接收端。
其发送数据如同发电报，对于一段需要发送的数据，每次都生成一个“数据包”实例，然后发送出去。

常用类：

1. InetAddress
2. DatagramSocket
3. DatagramPacket

TCP 和UDP 的区别

UDP 程序比TCP 程序相对简单，每次的数据传输都是独立的，TCP 的数据传输是连续的。

TCP 像生活中的电话，在建立连接后，数据传输的线路就确定了，不能更改了。
UDP 像生活中的信件，数据的目的是保持在DatagramPacket 实例中。DatagramPacket 像快递员，只负责将数据传输到目的地，没有固定的连接。

TCP 主要用来传出大量的、有序的数据。比如文件。
UDP 用来传输少量的、对顺序没有严格要求的数据。比如聊天、视频信息。
后者虽然是一种不可靠的网络协议，但其具有速度优势，而TCP 更安全可靠一些，但会有额外的系统开销。
